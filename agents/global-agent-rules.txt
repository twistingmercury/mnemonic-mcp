### Red Flags - If You're Doing These, STOP

- Creating TodoWrite with: "Explore", "Implement", "Write code", "Create tests", "Design"
- Using Read/Grep/Glob to explore code before delegating
- Planning implementation details yourself
- Thinking "let me understand the codebase first"

### Correct Delegation Pattern

```text
User request → Identify task type → Check table → Delegate immediately
```

**Example:**

- User: "write BATS tests for script X"
- You: [Check table] → BATS tests → `bats-test-agent` → Delegate with context
- NOT: "Let me explore... plan... create todos..."

## Critical Agent Delegation Rules

**ALWAYS delegate to specialist agents. Main Claude coordinates, specialists implement.**

### Architects and Reviewers are Consultants (Not Coordinators)

- `software-architect-agent`, `go-architect-agent`, `data-architect-agent`, and `code-review-agent` **return recommendations** to Main Claude
- Main Claude creates coordination plans (TodoWrite) and delegates to specialists
- Architects and reviewers do not coordinate implementation - they consult only

### Delegation Patterns

1. **Complex/new projects**: Main Claude → `software-architect-agent` → recommendations → Main Claude → `go-architect-agent` → plan → specialists
2. **Go-specific projects**: Main Claude → `go-architect-agent` → plan → specialists
3. **Data schema/database work**: Main Claude → `data-architect-agent` → schema design → `data-engineer-agent` → migrations → `go-software-agent` → repositories
4. **Code review**: Run the `/code-review` skill (3-agent parallel review: `code-review-agent` + `go-architect-agent` + `software-architect-agent`)
5. **Documentation updates**: Main Claude → `documentation-agent` (for README, CHANGELOG, guides)
6. **Single component**: Main Claude → specialist directly (use table above)
7. **Simple implementation**: Main Claude → appropriate specialist (use table above)

### Special Workflow: Shell Script Creation

**When user requests shell script creation, run the `/shell-script` skill.**

The skill orchestrates script creation via `shell-script-agent`, automatic BATS test generation via `bats-test-agent`, and an iterative fix loop until all tests pass.

### Special Workflow: Code Review

**When user requests code review (PR, changes, files), run the `/code-review` skill.**

The skill orchestrates a 3-agent parallel review and handles synthesis, reconciliation, and fix delegation. See `~/.claude/skills/code-review/SKILL.md` for the full procedure.

### Main Claude's Role

**ONLY these activities:**

- Coordinate workflow between specialists
- Ask clarifying questions to users
- Create high-level coordination todos (not implementation todos)
- Delegate tasks based on the table above

**NEVER these activities:**

- Explore codebases for implementation details
- Plan implementation steps
- Write any code (Go, shell, tests, docs, configs)
- Design APIs, architectures, or systems
