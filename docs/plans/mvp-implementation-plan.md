# MVP Implementation Plan

## High level plan

**IMPORTANT: NON-NEGOTIABLES**:

Before the next step can begin:

- Each step must pass testing as applicable. Agent: The user, coding agent(s) as needed
- the user has reviewed and commited the changes. Agent: The user
- The CI build has been ran locally and worked successfully. The user

Before the next phase can begin:

- Architectural review: validate deliverables against requirements and any issues fixed. Agents: software architect & go architect
- Update any docs as needed, i.e., README, CHANGELOG, architecture and design docs. Agents: go software agent & documentation agent
- The code review agent has reviewed the changes, the review documented, and any finding have been `DISMISSED`, `DEFERRED`, or `FIXED: {GIT COMMIT}`. Agents: code review agent, documentation agent, other agents as needed

## Vertical Slice Approach

This plan uses vertical slices for database implementation to deliver working functionality end-to-end:

- **Each slice completes one functional area**: schema design, migrations, repository implementation, and tests
- **Catches integration issues early**: Working code at each phase reveals problems immediately
- **Reduces context switching**: All work for one feature area happens together
- **PostgreSQL and PGVector combined**: PGVector is an extension, not a separate database
- **Neo4j is last**: System works without Neo4j (best-effort sync), so it comes after core PostgreSQL slices

## Phase Summary

This plan organizes database work into vertical slices, where each slice delivers complete end-to-end functionality:

| Phase Group            | Phases | Description                                                                                                                                                                                                                                             |
| ---------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Infrastructure**     | 1-4    | CI/CD, configuration, observability, and local development environment                                                                                                                                                                                  |
| **Test Foundation**    | 5      | Test infrastructure setup (helpers, fixtures, database utilities)                                                                                                                                                                                       |
| **PostgreSQL Slices**  | 6-8    | Three vertical slices implementing PostgreSQL functionality end-to-end: Agent slice (phase 6), Pattern slice with PGVector (phase 7), Rules & Jobs slice (phase 8). Each includes schema design, migrations, repository implementation, and unit tests. |
| **Migrations CI/CD**   | 9      | Independent CI/CD pipeline for database migrations                                                                                                                                                                                                      |
| **Neo4j Slice**        | 10     | Neo4j functionality end-to-end: schema design, constraints/indexes, GraphRepository, unit tests. Best-effort sync means system works without Neo4j.                                                                                                     |
| **Core Logic**         | 11-14  | Routing engine, API endpoints for route, patterns, and agents                                                                                                                                                                                           |
| **Pattern Enrichment** | 15-19  | Job queue, background worker, OpenAI integrations, Neo4j sync                                                                                                                                                                                           |
| **Production Deploy**  | 20-21  | Kubernetes manifests and Envoy proxy configuration                                                                                                                                                                                                      |
| **End-to-End Testing** | 22-24  | E2E tests for routing, pattern enrichment, and authentication                                                                                                                                                                                           |
| **Final Validation**   | 25     | Integration testing, performance validation, documentation review, architectural review                                                                                                                                                                 |

Key principles of vertical slices:

- **Deliver working functionality immediately**: Each slice produces testable, working code
- **Catch integration issues early**: Tests run against actual database schemas and repositories
- **Reduce context switching**: All work for one functional area happens together
- **PostgreSQL + PGVector combined**: PGVector is a PostgreSQL extension, not a separate database
- **Neo4j is last**: System works without Neo4j (best-effort sync), so it comes after core PostgreSQL slices

## Success Criteria Reference

From [MVP Scope](./mvp-scope.md) (Mnemonic-focused):

- Mnemonic routes requests to appropriate handlers via REST API
- Patterns are accessible via `/v1/api/patterns` endpoint
- Routing decisions are deterministic and reproducible
- Pattern enrichment pipeline processes and stores embeddings
