## Patterns to Document

1. **`RespondProblem` helper pattern**: Implementations of RFC 7807 with Gin must use a dedicated helper that sets `Content-Type: application/problem+json` before writing the body. `c.JSON()` sets `Content-Type: application/json` unconditionally and cannot be used for error responses without wrapper intervention.

2. **`X-Request-ID` generate-or-echo middleware**: The pattern of generating a UUID v4 when the header is absent (and echoing it when present) is standard for correlation headers. The generated value must be stored in Gin context so downstream helpers (like `RespondError`) read from context rather than re-reading the now-set header, avoiding a subtle order-of-operations dependency.

3. **Enum validation over empty check**: Fields constrained by a spec enum (such as `model`) must be validated against the allowed values, not merely checked for non-empty. Define the allowed values as a package-level `var` slice and validate with `slices.Contains`.
